# План реализации глобального модального окна

## 1. Подготовка компонентов

- [ ] Создать компонент-обертку `CreateBoardModal` (или использовать существующий `BoardFormDialog` с адаптацией) для использования внутри `ModalHost`.
  - Компонент должен подключаться к стору (использовать хук `useBoards`) для выполнения действия создания доски.
  - Компонент должен использовать хук `useModal` для закрытия модального окна.

## 2. Обновление ModalHost

- [ ] В файле `src/components/ModalHost/ModalHost.tsx`:
  - Импортировать необходимые компоненты (например, `CreateBoardModal`).
  - Заполнить `componentMap` соответствием `modalType` -> `Component`.
  - Реализовать рендеринг компонента из `componentMap` внутри `Dialog`.
  - Передать необходимые пропсы (например, `payload` из стора).
  - Настроить обработку закрытия окна (`onClose`).

## 3. Рефакторинг Boards (переход на глобальное модальное окно)

- [ ] В файле `src/components/Boards/Boards.tsx`:
  - Удалить локальное состояние управления диалогом (`useState` для `isOpen`, `mode`).
  - Подключить хук `useModal`.
  - Изменить обработчик создания доски так, чтобы он вызывал `modalOpen({ modalType: 'createBoard' })`.
- [ ] В файле `src/components/Boards/BoardsView.tsx`:
  - Удалить рендеринг `Dialog` и `BoardFormDialog`.
  - Удалить пропсы, связанные с локальным диалогом.

## 4. Проверка и тестирование

- [ ] Убедиться, что нажатие на кнопку создания доски открывает глобальное модальное окно.
- [ ] Убедиться, что создание доски работает корректно.
- [ ] Убедиться, что модальное окно закрывается корректно.
