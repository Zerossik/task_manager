# Архитектурные правила: Переход на Feature-Sliced Design (FSD)

Ты должен строго следовать этим правилам при генерации кода, рефакторинге и создании файлов. Мы находимся в процессе плавного перехода на архитектуру FSD.

## 1. Страницы и Роутинг (Pages)

- **Правило:** Все НОВЫЕ страницы, роуты или компоненты уровня экрана должны создаваться исключительно в директории `src/pages/`.
- **Структура:** Каждая страница должна находиться в своей отдельной папке. Внутри папки обязателен файл `index.ts` (Public API) для чистых экспортов наружу.

## 2. Общие UI-компоненты (Слой Shared)

- **Правило:** Любой переиспользуемый, атомарный или чисто презентационный компонент должен находиться в директории `src/shared/ui/` и экспортироваться из неё.
- **Интеграция с Material UI (MUI):**
  - **СТРОГО ЗАПРЕЩЕНО** импортировать компоненты напрямую из `@mui/material` внутри страниц, фич или виджетов.
  - Каждый используемый компонент MUI должен быть "пропущен" через наш слой shared-ui (своего рода прокси-экспорт).
  - Если нужного компонента MUI еще нет в `src/shared/ui/`, ты обязан СНАЧАЛА создать для него папку и реэкспорт в `src/shared/ui/`, и только ПОТОМ использовать его на странице.
